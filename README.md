Описание решения

1. **Запуск**:
   python3 fastq_stats.py

2. **Алгоритм**:
   - Читаем файл "reads.fastq" построчно.
   - Нужные последовательности находятся в строках с индексами 1, 5, 9... (каждая 4-я строка).
   - Подсчет статистики:
     - Количество прочтений: увеличиваем счетчик для каждой 4-й строки.
     - Длины (мин, ср, макс): вычисляем длину строки, обновляем min/max, суммируем общую длину.
     - GC-состав: считаем количество букв 'G' и 'C' во всех последовательностях, делим на общую длину всех последовательностей.

3. **Качество**:
   - Качество закодировано в строках с индексами 3, 7, 11...
   - Кодировка Phred+33: значение = `ord(символ) - 33`.
   - Для средней оценки на 10-й позиции берем 10-й символ (индекс 9) каждой строки качества и усредняем значения.

4. **Тримминг, Sliding Window**:
   - Используется метод скользящего окна (Sliding Window).
   - Размер окна: 5 оснований.
   - Порог качества: 30 (среднее качество в окне должно быть > 30).
   - Алгоритм:
     - Идем окном по качеству от начала к концу.
     - Если среднее качество в окне падает ниже порога (сумма <= 150), обрезаем чтение по этой позиции.
     - Дополнительно убираем с конца основания с качеством < 30 (backtracking).
     - Если после обрезки длина стала 0, считаем чтение удаленным.

5. **Фильтрация**:
   - Среди оставшихся после тримминга прочтений считаем только те, у которых длина >= 60.

6. **Результаты работы скрипта**:
   - Общее число прочтений: 239366
   - Диапазон длин (мин/макс): 17 / 151
   - Средняя длина: 148
   - GC-состав: 42.94%
   - Среднее качество на позиции 10: 36
   
   После фильтрации:
   - Удалено прочтений (длина стала 0): 15028
   - Минимальная длина оставшихся: 3
   - Средняя длина оставшихся: 106
   - Максимальная длина оставшихся: 151
   - Оставшихся прочтений с длиной >= 60: 165315
